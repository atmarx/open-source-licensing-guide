<!-- Glossary abbreviations - auto-appended to all pages via pymdownx.snippets -->

*[OSI]: Open Source Initiative - organization that reviews and approves licenses as "open source"
*[FSF]: Free Software Foundation - organization that advocates for free software and maintains the GPL
*[SPDX]: Software Package Data Exchange - standard format for communicating license information
*[copyleft]: A licensing approach requiring derivative works to use the same license
*[permissive license]: A license that allows use with minimal restrictions, including in proprietary software
*[derivative work]: A new work based on or derived from an existing copyrighted work
*[distribution]: The act of sharing or providing software to others
*[attribution]: Credit given to the original creator as required by a license
*[patent grant]: License provision giving users rights to any patents held by contributors
*[viral license]: Informal term for copyleft licenses that "spread" to derivative works
*[source code]: Human-readable form of software, as opposed to compiled binary code
*[binary]: Compiled, machine-readable form of software
*[proprietary]: Software with restricted rights, typically closed-source and commercially licensed
*[public domain]: Works not protected by copyright, free for any use without restriction
*[sublicense]: The right to grant license rights to others
*[notice]: Required text that must be included when redistributing licensed software
*[disclaimer]: Statement limiting liability, typically required to be preserved
*[share-alike]: License term requiring derivatives to use the same or compatible license
*[NonCommercial]: License restriction prohibiting use for commercial purposes
*[NoDerivatives]: License restriction prohibiting creation of derivative works
*[weak copyleft]: Copyleft that only applies to modifications of the original files, not the larger work
*[strong copyleft]: Copyleft that applies to the entire combined work, not just modified files
*[license compatibility]: Whether code under different licenses can be legally combined
*[dual licensing]: Offering software under two different licenses simultaneously
*[contributor]: Someone who adds code or content to a project
*[upstream]: The original project from which a fork or derivative is created
*[downstream]: Projects or users that build upon or use an upstream project
*[libre]: Free as in freedom - referring to rights and permissions, not price
*[gratis]: Free as in cost - referring to price, not rights
*[OSI-approved]: A license officially recognized by the Open Source Initiative
*[FOSS]: Free and Open Source Software
*[FLOSS]: Free/Libre and Open Source Software - emphasizes the freedom aspect
